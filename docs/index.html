Zero Transpiler

[ View on GitHub ]

Why Zero?

  • Hallucination-proof: Designed to constrain LLMs and avoid syntax or semantic hallucinations.
  • Immediate Semantic Feedback via Go: Transpiles to Go, allowing for immediate type checking and robust semantic validation.
  • Abstraction Constraints: Simplifies concepts to keep the AI-generated code predictable and maintainable.
  • LLM-Native Primitives: Built-in llm_generate, fuzzy_cast, and assert_semantic to trivially utilize, orchestrate, and enforce constraints on other AIs directly from the language.
  • AST-Level Semantic Patching: Allows AIs to surgically update specific functions via a (patch ...) node without perfectly rewriting large files.
  • Native Automation & String Suite: Built-in support for File I/O (read_file, write_file, mkdir), subprocess execution (exec), CLI argument parsing (cli_args), string manipulation (str_split, str_join, regex_match), and type conversion (to_int, to_float, to_string).
  • Advanced Control Flow: Provides primitives tailored for AI parsers and state machines, including while loops, match pattern matching, and set variable mutations.
  • Safe Concurrency & Guardrails: Easily spawn background routines (spawn), make HTTP requests (fetch), or enforce AI API call safety using rate_limit and retry blocks.
  • Database & Persistence: Native SQL connection and query primitives (db_connect, sql_query) that transpile directly to Go's standard database/sql package.
  • Modularity & Imports: Support for file inclusion via include and standard Go package imports via import.
  • Collections & Typed Structs: Strong typing with struct (including direct field access on parsed JSON), mutable list mutation (append), map management (map_set, map_delete), and random-access reads (map_get, list_get).
  • Security & Middleware: Protect routes with HTTP middleware, pipeline control (next), and environment variable reads (env).
  • Implicit Context Threading: Built-in (with_context ...) macro that implicitly injects variables into function calls, reducing the cognitive load for AIs to remember to pass state like request objects and database connections down the call stack.
  • Native Unit Test Blocks: Accelerates Test-Driven Development (TDD) for AIs with a native (test "description" ...) block that seamlessly compiles to Go test files.
  • Compiles to JavaScript too: Swap the root block to (web_app ...) to target the browser with the same grammar — DOM primitives (dom_query, on_event, set_text) plus every existing logic primitive, unchanged.
  • Built-in Observability: Every generated program auto-injects call tracing (telemetry.jsonl) and crash-state serialization (crash.json with a full stack trace on panic) with zero extra syntax, plus a standalone Python observer agent (observer.py) that watches for anomalies live.
  • Go Generics: (type_param T) on a defun generates a real Go generic function instead of falling back to any and runtime type assertions.
  • Direct AST Execution: -run interprets a cli_app script directly — no Go text ever generated, no go build/go run invoked — Phase 1 toward Zero's eventual goal of bypassing text-based codegen entirely (see the project's roadmap).

Installation & Setup

To use the AI Orchestrator to generate Zero code, you need a few tools:

  • Go 1.20+ (To compile the transpiled code)
  • Python 3.10+ (For the orchestrator script)
  • Ollama (To run local LLMs like llama3)
git clone https://github.com/howlcipher/zero.git
cd zero
pip install outlines openai
ollama serve
ollama pull llama3

Generating Code with AI (Orchestrator)

We provide an orchestrator script (orchestrator.py) that handles interaction with the LLM, strictly enforces syntax boundaries, and handles error feedback loops.

  1. Ensure Ollama is running and the llama3 model is available.
  2. Open orchestrator.py and modify the prompt variable to instruct the AI on what to build.
  3. Run the orchestrator: python orchestrator.py
  4. The AI will generate a `.zero` file. The orchestrator will automatically run the Go transpiler.
  5. If the transpiler encounters a semantic error, the orchestrator intercepts it and sends it back to the AI for automatic self-correction.
  6. Once successful, it compiles the Go binary and executes the application.

How much does writing Zero actually cost?

Not a traditional compile/runtime benchmark — this measures the LLM write cost (wall-clock time and token count) of producing a correct, compiler-verified solution to the same task prompt in Zero vs. Go, Python, Node.js, C#, and Java. Full methodology, per-task breakdowns, and the two transpiler bugs discovered while building it are in the full write-up.

Zero50.1295

Java21.7438

Zero wins clearly on its designed niche (HTTP/JSON handlers) and is mid-pack on total tokens — but it is not a uniform win across every program type, and its total write time here is inflated by real, now-fixed-in-the-backlog transpiler gaps found mid-benchmark.
