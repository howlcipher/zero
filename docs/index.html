Why Zero?

  • Hallucination-proof: Designed to constrain LLMs and avoid syntax or semantic hallucinations.
  • Immediate Semantic Feedback via Go: Transpiles to Go, allowing for immediate type checking and robust semantic validation.
  • Abstraction Constraints: Simplifies concepts to keep the AI-generated code predictable and maintainable.
  • LLM-Native Primitives: Built-in llm_generate to trivially utilize and orchestrate other AIs directly from the language.
  • AST-Level Semantic Patching: Allows AIs to surgically update specific functions via a (patch ...) node without perfectly rewriting large files.
  • Native Automation: Built-in support for File I/O (read_file, write_file, mkdir), subprocess execution (exec), CLI argument parsing (cli_args), and string manipulation suite.
  • Advanced Control Flow: Provides primitives tailored for AI parsers and state machines, including while loops, match pattern matching, and set variable mutations.
  • Safe Concurrency & Guardrails: Easily spawn background routines or enforce AI API call safety using rate_limit and retry blocks.
  • Implicit Context Threading: Built-in (with_context ...) macro that implicitly injects variables into function calls, reducing the cognitive load for AIs to remember to pass state like request objects and database connections down the call stack.
